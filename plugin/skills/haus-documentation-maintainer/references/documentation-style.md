# Documentation Style Guide

- Be direct and practical.
- Distinguish implemented vs planned.
- Prefer examples over abstract explanations.
- Use real command names from `src/cli.ts`.
- Keep docs Claude-native and repo-specific.
- Avoid marketing fluff.
- Avoid giant monolithic docs.
- Cross-link docs where useful.
- Keep `README.md` short.
