# ADR-0004: Deep comprehension feeds asset selection via deep-context.json

- **Status:** Accepted | **Date:** 2026-06-03

## Context

The scanner is a shallow, deterministic classifier — it gates which catalog assets get
recommended. Roles/patterns it cannot detect mean relevant skills are never applied. The
`writing-documentation` skill comprehends a repo far more deeply (wired modules, domain
patterns, runtime topology), but it is LLM-driven and cannot run in the headless paths
(CI, prepack, doctor, refresh) that recommendation must support.

## Decision

Add an enrichment feedback loop instead of deepening the scanner or running the skill
first:

1. The skill, after its deep scan, writes `.haus-workflow/deep-context.json`
   (`{ source, roles, stacks, patterns }`) — best-effort, LLM-authored.
2. `recommend` reads it defensively and merges its signals with `context-map.json`
   (union). Matches sourced from it are tagged `deep:…` for transparency.
3. A second `recommend`/`apply` pass installs skills the shallow scan missed.

When the file is absent, output is identical to the scanner-only pass — the headless
path is never broken, and determinism is preserved.

## Consequences

- Deep comprehension improves asset selection without an LLM in the headless path.
- Two context files coexist by design: `context-map.json` (shallow, deterministic,
  hash-tracked, regenerated by rescans) and `deep-context.json` (deep, skill-owned,
  survives rescans). They are not merged into one to avoid rescans clobbering enrichment.

## Alternatives considered

- **Write enrichment back into `context-map.json`** — rejected; a rescan/`refresh` regenerates it from code and would clobber the LLM enrichment.
- **Run the skill before the scanner** — rejected; recommendation must work headless, and the skill seeds from the scanner.
